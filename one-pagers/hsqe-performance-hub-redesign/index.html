HSQE Performance Hub redesign — pilot built, ready for review

The 23-page report is being rebuilt in place on the Ground Control design system, every number reconciled to the old pages. Sign-off on the four pilot pages locks the template for the remaining nineteen.

For: HSQE stakeholders  ·  From: Aaron Haydon  ·  July 2026
Where we are

The pilot slice is built and verified

We're rebuilding the HSQE Performance Hub page by page — same data, same numbers, a far clearer report. The work is template-driven and theme-first, done in place on the live .pbip under git. The semantic model is frozen: we add measures, we never touch the data. Foundations are done, the four pilot pages are built and verified in Power BI Desktop, and the one thing standing between us and the full rollout is stakeholder sign-off.

Foundations

One design system, no ad-hoc styling

The Ground Control theme, a shared component library, and a dedicated measure base — deltas, RAG flags, last-refreshed — now drive every page. Colours and fonts come from tokens, not judgement calls.

Pilot pages

Four pages, three archetypes

Events Tracker is the master template, joined by Time Study & AFR, the HSQE Overview scorecard, and the Main Menu hub. Between them they cover the layouts the other nineteen pages will reuse.

Parity

Every metric reconciles

Each rebuilt page is checked number-for-number against the page it replaces, so the redesign changes how it reads, not what it says.

The design system

Ground Control tokens, applied at dashboard density

Every page uses the same palette, type scale, and spacing. Canvas is 1920×1200 on a Warm Grey background — the extra 120px over the old 1080 canvas is reflow headroom to spread crowded rows, not dead space. Cards carry a 16px radius with dark-green tinted shadows, never pure black. Cera Pro is the typeface throughout — Bold for headings, Medium for overlines and labels, Regular for body.

Dark Green#294238 · primary brand, body text
Light Green#B2D235 · preferred accent, highlights
Mid Green#50B748 · secondary accent, success
Warm Grey#E6EBE3 · page and card surfaces
Orange#F57821 · warnings, used sparingly
Deep Orange#C64A0F · true error and RAG red
Neutral grey-green#7E8A80 · neutral state, tertiary text
Secondary wash#33534A · chart secondary series

RAG semantics

Good / on-target — Mid Green
Neutral / watch — grey-green
Bad / breach — Deep Orange, never pure red

Status is never colour-only: every RAG colour is paired with an icon or text label. There is no true amber token — the neutral grey-green covers the watch state, and Orange is reserved for genuine warnings.

Fixed along the way

Defects the old report had lived with

  • White-on-white text in Desktop — visuals bound to ThemeDataColor ColorId 0 resolved to the white background rather than the first data colour, leaving text invisible. All colour bindings now use literal hex
  • Clipped and empty textboxes — a systemic height rule (height ≥ ceil(fontSize × 4/3) + 26) fixed textboxes that clipped descenders or rendered as empty pills, applied across every page with a per-page reflow
  • Narrative text that drifted — AI-generated narrative visuals changed their wording on every refresh. They are replaced with deterministic DAX narrative measures: the same data now produces the same sentence, every time
  • Blank-date filters — date filters that silently dropped blank rows are corrected
  • Header polish — theme card backgrounds disabled on band textboxes and the Ground Control logo bound into the header band
Per-page definition of done
  • Theme and component library only — no ad-hoc colours or fonts
  • One clear focal visual — plus a KPI strip with deltas and RAG
  • Parity — every metric reconciles to the old page's numbers
  • Slicers synced — cross-filtering tuned
  • Tooltips under 40 characters — alt text present, no truncated labels
  • Within the performance budget
Working assumptions

In-place rebuild, not a parallel copy · model frozen, new measures only · Environment built native if the data is available · single-developer pace throughout. Each was flagged before Phase 0; say so now if any should change.

We deliberately built one page of each layout type before touching the rest. Every design decision — the header band, the KPI strip, the slicer bar, how RAG status is shown — gets made once, on the pilot, then rolled out mechanically. That's why sign-off matters: it locks the template, and everything after it is repetition rather than reinvention.
